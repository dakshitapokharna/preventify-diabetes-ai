2. Why WhatsApp Testing

The product's north-star delivery channel is WhatsApp. Testing on it early validates:

  • Message formatting in a real WhatsApp environment
  • Response latency end-to-end
  • Conversation flow outside a controlled web UI
  • Any edge cases in the adapter layer before production

3. Architecture — What Changes

Nothing in the core pipeline changes. A thin WhatsApp adapter layer is added on top:

User messages test WhatsApp number (personal phone)
  → Meta Cloud API receives message
  → POSTs to your /webhook endpoint on FastAPI
  → Webhook extracts phone number + message text
  → Feeds into existing Phase 1 → Phase 2 pipeline (unchanged)
  → Response sent back via Meta's send-message API
  → User receives reply on WhatsApp

5. Step-by-Step Setup

Step 1 — Expose Localhost with ngrok

  1. Sign up at ngrok.com (free)
  2. Download and install ngrok
  3. Run in terminal: ngrok http 8000  (replace 8000 with your FastAPI port)
  4. Copy the HTTPS URL it gives you — e.g. https://abc123.ngrok-free.app
  5. Keep this terminal open while testing
On the free tier, this URL changes every time you restart ngrok. You will need to update the Meta webhook URL each session.

Step 2 — Create App on Meta for Developers

  1. Go to developers.facebook.com
  2. Log in with your personal Facebook account
  3. Click My Apps → Create App
  4. Choose Other → Business
  5. Give it a name (e.g. PreventifyBot) → Create App

Step 3 — Add WhatsApp Product & Get Test Number

  1. On your app dashboard, find WhatsApp → click Set up
  2. Meta provides a test phone number automatically — no SIM or company number needed
  3. Under To, add your personal WhatsApp number as a test recipient
  4. Send a test message from the dashboard to verify delivery works

Step 4 — Configure Webhook

  1. Go to WhatsApp → Configuration → Webhook
  2. Set Webhook URL to your ngrok URL + /webhook
    Example: https://abc123.ngrok-free.app/webhook
  3. Set Verify Token to any string, e.g. preventify123
  4. Click Verify and Save
  5. Under Webhook Fields, subscribe to messages

Step 5 — Add /webhook Route to FastAPI

Add one new route to your existing FastAPI app. This route:

  • Handles Meta's one-time verification GET request (handshake)
  • Handles incoming POST requests (actual messages from users)
  • Extracts message text and sender phone number
  • Passes message into your existing Phase 1 → Phase 2 pipeline
  • Calls Meta's API to send the response back to the user

This is the only new code needed — everything else stays exactly as is.

Step 6 — Test End-to-End

  1. Make sure your FastAPI server is running locally
  2. Make sure ngrok is running and the URL is set in Meta dashboard
  3. From your personal WhatsApp, send a diabetes-related question to the test number
  4. Watch it flow through your pipeline and reply

7. After Testing — Path to Production

  1. Deploy FastAPI to a cloud server (Railway, Render, or a VPS) for a permanent HTTPS URL
  2. Replace ngrok URL with production URL in Meta dashboard
  3. Apply for a real WhatsApp Business phone number via Meta
  4. Submit message templates for approval if initiating conversations
  5. This comes after clinical sign-off (B2 + B4 blockers resolved)
